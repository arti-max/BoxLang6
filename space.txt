BoxLang6 - Язык программирования для 2х, 4х, 8ми, 16-ти битных эзотерических систем. (например x16-PRos или вообще системы на эмуляторах с собственными инструкциями). Т.е. нужно будет сделать модульность с помощью json/toon, чтобы можно было легко добавить свою архитектуру процессора, не меняя код компилятора. Должен собираться сразу в binary код, но для оладки можно сделать ещё промежуточный debug файл на ассемблере (опять-же для 16-ти бит использовать nasm, а для других ассемблеров писать json файл синтаксиса).

Нужно сделать так, чтобы в будущем можно было собрать этот язык программирования на себе-же и пихать его во все системы/вообще self-hosted как отдельная ОС. (для полной совместимости со всем железом). 

Функции в языке, которые должны быть:
1. box - объявление функции
box _start[] : num16 (
   ret 0; // _start - точка входа.
)
2. типы переменных - bit, bit2, bit4, char, num16, type[count], type*. (num32?)
num16 myVar: 16;
3. система указателей
4. namespace'ы.
namespace MySpace (
      box myFunc[char a] (
         // do smth.
      )
)
5. open - вызвать функцию
open myFunc[args];
open namespace::myFunc[args];
6. цикл while, for.
while [a < 5] (
   // do smth
)
7. asm вставки:
asm["mov", ax, x]; // автоматически на основе инстуркций в json матчит операнды и т.д., и создаёт бинарное представление.
8. вложенные переменные.
9. exit[] func (встроенная функция завершения программы, генерирует пролог с bp точки входа, восстанавливая его в любой точке кода)
10. shelf (аналог struct)
shelf MyShelf (
    num16 varA: 0;
    char b;
)
11. $include - импорт файлов
$include <stdlib> - импорт файлов из стандартной библиотеки, т.е. из папки std в папке компилятора
$include "filename.box" - импорт из файла в директории проекта.
12. $use name - использовать json файл конфигурации системы с syscall'ами и настройками под систему, например парсинг аргументов из командной строки, если $use не указан, то это значит, что пишется на голое железо self-hosted программа.
$use x16pros
$use govnos


структура файлов:

boxlang/
├── compiler/
│   ├── lexer.py          # Токенизатор
│   ├── parser.py         # Парсер → AST
│   ├── ast_nodes.py      # Узлы AST
│   ├── semantic.py       # Семантический анализ
│   ├── codegen.py        # Генерация кода (asm/binary) (не используется, код генерируется в binary_target)
│   └── optimizer.py      # Оптимизации (позже)
│
├── arch/                 # JSON-конфиги архитектур
│   ├── x16.json          # x86-16
│   ├── x8.json           # 8-битная архитектура (например 6502 MOS)
│   └── schema.json       # Схема валидации конфига
│
├── std/                  # Стандартная библиотека
│   ├── stdlib.box
│   └── io.box
│
├── targets/
│   ├── binary_target.py  # Прямая генерация бинарника
│   └── base_target.py    # Базовый класс таргета
│
├── debug/
│   └── asm_dumper.py     # Debug-дамп в asm (не нужен)
│
├── main.py               # Точка входа компилятора
└── config.py             # Глобальные настройки (не используется)

BoxLang Debugger — план на будущее
Визуализация AST — дерево с плавными кривыми (например через pygame или веб-интерфейс на FastAPI + D3.js), каждый узел кликабельный, подсвечивает соответствующую строку кода.

Realtime панели:
| Панель      | Содержимое                                        |
| ----------- | ------------------------------------------------- |
| AST Tree    | Живое дерево узлов с анимацией при парсинге       |
| Stack Trace | Текущий call stack, значения переменных           |
| ASM View    | Генерируемый ассемблер — строка за строкой        |
| Hex View    | Машинный код в hex + подсветка текущей инструкции |
Стек технологий — либо всё в Python (pygame / tkinter), либо компилятор остаётся на Python, а дебаггер — лёгкий веб-UI через FastAPI + WebSocket для realtime обновлений. Второй вариант красивее и проще рисовать кривые.
